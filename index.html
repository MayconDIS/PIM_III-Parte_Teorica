RESUMO

O presente trabalho descreve o desenvolvimento do projeto Nex_TI, um sistema web voltado ao nivelamento acadêmico e apoio pedagógico de estudantes ingressantes na área de Tecnologia da Informação. O objetivo do sistema é mitigar as dificuldades de aprendizagem de conceitos técnicos iniciais e reduzir a evasão escolar por meio de uma plataforma baseada em estudo ativo. A arquitetura de software proposta utiliza o algoritmo de repetição espaçada SuperMemo-2 (SM-2) integrado a um sistema gamificado de incentivos. Metodologicamente, a engenharia do projeto foi estruturada sob os preceitos do desenvolvimento ágil (Scrum e Kanban), compreendendo: a especificação de requisitos funcionais e não funcionais; a modelagem conceitual e física de banco de dados relacional (SQL Server); o projeto lógico orientado a objetos em C# (.NET 10); e o protótipo de interface responsiva em HTML5 e CSS com técnicas de acessibilidade (WAI-ARIA). Os resultados demonstram a consistência lógica e a viabilidade da integração entre as disciplinas do semestre através de diagramas UML e protótipos interativos no Figma, fornecendo a base técnica para a consolidação de um MVP (Produto Mínimo Viável) em conformidade com a LGPD.


Palavras-chave: Nivelamento Acadêmico; Repetição Espaçada; Algoritmo SM-2; Modelagem UML; Banco de Dados.

ABSTRACT

This paper describes the development of the Nex_TI project, a web-based system designed for academic leveling and pedagogical support for incoming Information Technology students. The system aims to mitigate difficulties in learning initial technical concepts and reduce dropout rates through a platform based on active learning. The proposed software architecture employs the SuperMemo-2 (SM-2) spaced repetition algorithm integrated with a gamified incentive system. Methodologically, the engineering of the project was structured under the principles of agile development (Scrum and Kanban), comprising: the specification of functional and non-functional requirements; conceptual and physical relational database modeling (SQL Server); object-oriented logical design in C# (.NET 10); and a responsive interface prototype in HTML5 and CSS incorporating accessibility standards (WAI-ARIA). The results demonstrate the logical consistency and feasibility of integrating the semester's disciplines using UML diagrams and interactive Figma prototypes, establishing the technical foundation for a Minimum Viable Product (MVP) in compliance with the General Data Protection Regulation (LGPD).


Keywords: Academic Leveling; Spaced Repetition; SM-2 Algorithm; UML Modeling; Databases.

1 INTRODUÇÃO

O ingresso no ensino superior na área de Tecnologia da Informação frequentemente expõe os estudantes a uma quantidade expressiva de novos termos técnicos e jargões. A falta de familiaridade com esses conceitos fundamentais é apontada como um dos fatores que contribuem para o desinteresse acadêmico e as taxas de evasão em calouros. Para mitigar esse cenário, torna-se relevante a criação de ferramentas pedagógicas que facilitem a memorização e o estudo ativo contínuo de forma acessível.

O projeto Nex_TI propõe o desenvolvimento de uma plataforma web de nivelamento baseada em flashcards e no algoritmo de repetição espaçada SuperMemo-2 (SM-2). Este trabalho documenta a engenharia do sistema, compreendendo a gestão ágil de requisitos, a modelagem de banco de dados relacional SQL Server e a arquitetura de programação orientada a objetos em C#. O projeto da interface web é estruturado a partir de protótipos de alta fidelidade e layouts responsivos em HTML e CSS, pautando-se pelas diretrizes de acessibilidade e segurança da informação aplicadas ao contexto de um MVP.

1.1 Objetivo Geral

Desenvolver e documentar uma plataforma web gamificada baseada em algoritmos de repetição espaçada, entregando uma arquitetura escalável, modelagem de banco de dados estruturada e protótipos focados no engajamento e retenção de conhecimento dos usuários.

1.2 Objetivos Específicos

  • Modelar a estrutura de dados relacional e implementar a lógica via Orientação a Objetos (C#).
  • Gerenciar o ciclo de vida do software utilizando metodologias ágeis (Scrum e Kanban).
  • Projetar interfaces web responsivas com HTML Semântico, CSS Grid/Flexbox e foco em acessibilidade.
  • Garantir o tratamento seguro de dados em conformidade com a LGPD.

O desenvolvimento da Nex_TI baseia-se em metodologias de engajamento comprovadas por grandes plataformas educacionais de mercado, que atestam o aumento da taxa de conclusão por meio de trilhas de conhecimento interativas. O diferencial do sistema reside na aplicação direta de algoritmos de repetição para automatizar os fluxos de revisão temporal, transformando a aprendizagem passiva em um processo quantificável e avaliativo.

2.3 Etapa 3 – Modelagem de Banco de Dados e NoSQL

Para a escolha da tecnologia de armazenamento de dados, o projeto comparou o modelo tradicional de tabelas (relacional) com modelos mais flexíveis (não-relacionais, conhecidos como NoSQL). O banco de dados implementado no projeto é exclusivamente o Microsoft SQL Server. A justificativa técnica para essa escolha baseia-se na necessidade de garantir que as transações ocorram de forma totalmente segura. Pense no saldo de moedas e na pontuação do aluno como uma transferência bancária: a operação deve acontecer por completo ou ser totalmente cancelada caso ocorra alguma falha no caminho. Isso impede que o saldo do usuário seja duplicado ou corrompido em acessos simultâneos.

Além disso, a estrutura de tabelas interligadas do SQL Server garante que, se uma pergunta do simulado for removida, todas as suas alternativas de resposta sejam apagadas automaticamente do sistema, evitando que dados incorretos ou incompletos permaneçam na base de dados. Em relação às tecnologias NoSQL, embora apresentem grande flexibilidade e velocidade para lidar com volumes massivos de dados não-estruturados, elas não foram adotadas na implementação prática deste projeto. Isso ocorre porque os bancos NoSQL priorizam a rapidez de gravação e a distribuição de dados em rede em detrimento de relacionamentos rígidos e garantias transacionais perfeitas, características que são essenciais para as regras de pontuação e consistência do nosso sistema de nivelamento.

2.4 Etapa 4 – Desenvolvimento do sistema com POO (C#)

A programação do sistema foi desenvolvida na linguagem C# baseando-se nos quatro princípios da Programação Orientada a Objetos:

Abstração: Consiste em simplificar um objeto real para o sistema. Criamos o modelo básico Usuario (Figura 7) que armazena apenas as informações que o sistema realmente precisa para funcionar, como o nome, e-mail e senha de login do usuário, ocultando as engrenagens mais complexas do servidor.

Encapsulamento: Funciona como uma proteção de segurança, semelhante a um cofre digital. Na classe do Aluno (Figura 8), os atributos de experiência e moedas possuem escrita trancada de forma que nenhuma parte externa do código consiga alterar o saldo diretamente. O saldo só pode ser modificado através de ações permitidas e controladas pelo próprio sistema.

Herança: Permite que novos modelos aproveitem as características de modelos já existentes. Criamos perfis específicos para Aluno e Tutor (Figura 8) que aproveitam e estendem automaticamente a estrutura comum de cadastro definida no modelo de Usuario, evitando reescrever código desnecessariamente.

Polimorfismo: Dá flexibilidade ao sistema ao permitir que uma mesma instrução se comporte de formas diferentes. Por exemplo, o comando de iniciar estudos se adapta de forma transparente caso a ação esteja sendo feita em um flashcard comum ou em uma atividade de simulado, mantendo a estrutura flexível.

Figura 7 – Classe Base em C#

2.7 Etapa 7 – Machine Learning e Análise de Dados

A plataforma Nex_TI utiliza uma técnica para ajudar os alunos a reterem o conhecimento de forma mais eficiente, calculando o melhor momento para revisar cada assunto e evitando que o conteúdo seja esquecido naturalmente. O processo funciona de forma personalizada com cartões de estudo (flashcards): após responder a uma pergunta, o aluno indica o quanto foi fácil responder.

O sistema define o tempo de espera do cartão a partir de duas regras simples:

1. Complexidade: Se o aluno achou o assunto fácil, o tempo para a próxima revisão aumenta. Se achou difícil, o cartão retorna rapidamente para a rotina diária até que seja fixado.

2. Tempo de Revisão: A próxima exibição é agendada em dias. O intervalo aumenta progressivamente a cada acerto, garantindo que o estudante revise mais o que ainda não domina totalmente.

Todo esse histórico fica registrado no banco de dados, servindo como base para que futuramente inteligências artificiais analisem o engajamento e alertem tutores caso algum estudante apresente dificuldades constantes.
